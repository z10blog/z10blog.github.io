git 基本操作

git 是一个开源的版本控制系统
master 主分支
git reset HEAD file命令既可以回退版本，也可以把暂存区的修改回退到工作区。当我们用HEAD时，表示最新的版本。
rm test.txt 命令只是从工作区删除了文件，仓库中仍然存在这个文件，如果要从仓库中删除这个文件可以使用git rm test.txt 命令，另外如果是误删了，我们可以使用git checkout – test.txt 命令把文件从仓库中恢复到工作区。
HEAD 表示当前版本，HEAD^表示上一个版本，HEAD^^…，也可以写成HEAD~100
常用命令：

1. git init 初始化一个git 仓库
2. git add readme.txt 往仓库暂存区里添加文件
3. git commit -m "add commit des" 把所有暂存区内容提交到当前分支
4. git status 查看仓库状态
5. git diff readme.txt 查看文件文件修改了那些内容
6. git log 查看提交历史 --pretty=oneline 之显示commit id 和 提交说明
7. git reset --hard HEAD^ | git reset --hard 323423 指定comit id
7. git reflog 用来打印出历史命令记录
http://www.liaoxuefeng.com/files/attachments/001384907702917346729e9afbf4127b6dfbae9207af016000/0
8. git diff HEAD -- readme.txt 查看工作区和版本库里面最新版本的区别
9. git checkout -- readme.txt 丢弃工作区中的修改

** 远程仓库 remote 远程
$git remote add origin git@github.com:xxxxxx/xxxxx.git
origin 是仓库的名称
$ git push -u origin master
把本地库的内容推送到远程，用git push命令，实际上是把当前分支master推送到远程。

由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。如下：
$ git push origin master

** 从远程仓库克隆
当我们从零开发时先创建一个远程仓库。
让后使用git clone 克隆一个本地仓库
$ git clone git@github.com:xxxxxx/xxxx.git
$ cd xxxx
使用https除了速度慢以外，还有个最大的麻烦是每次推送都必须输入口令，但是在某些只开放http端口的公司内部就无法使用ssh协议而只能用https。

** 分支 branch
分支在实际中有什么用呢？假设你准备开发一个新功能，但是需要两周才能完成，第一周你写了50%的代码，如果立刻提交，由于代码还没写完，不完整的代码库会导致别人不能干活了。如果等代码全部写完再一次提交，又存在丢失每天进度的巨大风险。

$ git branch dev 创建分支
$ git checkout -b dev 创建分支并切换过去
$ git branch 查看当前分支
$ git merge dev 把dev分支合并到master分支上,必须切换到master上
$ git branch -d dev 删除dev分支，合并之后再删除分支dev

** 冲突
当master 分支test.txt 文件与 dev 分支test文件不同时，要执行合并命令是就会出现冲突，这时我们是不能提交的，只能解决冲突之后才能提交，查看冲突的文件test.txt <<<<<=======>>>>>>是git标记出不同分支的内容。
修改过冲突文件之后从新git add test.txt git commit -m “xxx”就可以了。
最后要删除dev分支

用git log –graph命令可以看到分支合并图

合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。

** Bug分支
git stash 把当前工作现场存储起来，等以后恢复现场后继续工作,这是git status查看工作区就是干净的
$ git stash list 查看有哪些stash
$ git stash apply 恢复存储的工作，并没有删除stash
$ git stash drop 删除stash
$ git stash pop 恢复的同时删除stash

$git stash apply stash@{0} 删除指定stash

** git branch -D feature-vulcan 在没有合并时强制删除该分支

$ git remote 查看远程分支
$ git remote -v 想看远程仓库详情

** 推送分支
推送分支，就是把该分支上的所有本地提交推送到远程库。推送时，要指定本地分支，这样，Git就会把该分支推送到远程库对应的远程分支上：
$git push origin master
这样就把本地master分支推送到远程仓库origin上了
$git push origin dev 这样就把本地dev分支推送到远程origin分支了

但是，并不是一定要把本地分支往远程推送，那么，哪些分支需要推送，哪些不需要呢？

master分支是主分支，因此要时刻与远程同步；

dev分支是开发分支，团队所有成员都需要在上面工作，所以也需要与远程同步；

bug分支只用于在本地修复bug，就没必要推到远程了，除非老板要看看你每周到底修复了几个bug；

feature分支是否推到远程，取决于你是否和你的小伙伴合作在上面开发。

总之，就是在Git中，分支完全可以在本地自己藏着玩，是否推送，视你的心情而定！

** 抓取分支
当你的小伙伴从远程库clone时，默认情况下，你的小伙伴只能看到本地的master分支。

现在，你的小伙伴要在dev分支上开发，就必须创建远程origin的dev分支到本地，于是他用这个命令创建本地dev分支：
$git checkout -b dev origin/dev
现在，他就可以在dev上继续修改，然后，时不时地把dev分支push到远程：
$ git push origin dev

** tag
git tag v1.0 创建tag
git tag 查看tag
$git tag v0.9 commitId
git show 参考tag

git tag -d v0.1 删除tag
git push origin v1.0 推送tag到远程
git push origin –tags 推送全部tag到远程
git push origin :refs/tags/可以删除一个远程标签。

Python 函数认识

每种编程语言中都有函数的功能，函数的作用是什么？

  1. 把程序分成不同的功能块，每个函数代表了一个功能
  2. 封装逻辑实现，我们调用设计好的函数只需要输入相关的参数即可得到我们需要的结果，从而实现程序的重复使用。

要点：

  1. 默认参数必须指向不可变对象，否则程序运行时会有逻辑错误。
  2. 定义可变形参要在参数名前加’*’
  3. 调用可变参数的函数时可以直接输入多个参数，或者直接传入一个tuple，如果直接传入一个含有多个对象的tuple时需要在对象前加’*’
  4. 命名关键字参数：
    1. 命名关键字参数需要一个特殊的分隔符,后面的参数被视为命名关键字参数。
    2. 如：def person(name, age, , city, job): city和job在后边，所以就是命名关键字参数。
  5. 如果一个函数中已经有了一个可变参数，后面跟着的命名关键字参数就不需要分隔符*了
  6. 命名关键字参数必须传入参数名
  7. 命名关键字参数可以有默认值，如果有默认值则可以省略，使用默认值。
  8. Python中的函数可以混合各种参数，但是请注意，参数定义的顺序必须是：必选参数，默认参数，可变参数，命名关键字参数和关键字参数。
  9. *args 是可变参数，args接受的是一个tuple
  10. **kw 是关键字参数，kw接受的是一个dict

Python 数据类型和变量

编程就是为了处理各种数据，当然每种编程语言中都有多种类型的数据。每种编程语言都自带的内置数据类型都无法满足我们的需要，但是每种编程语言都提供了让我们创建自己的数据类型的机制。刚学首先要掌握的就是最基本的数据类型，慢慢的接触更多的和自己创建新的数据类型。

type() 认识

type() 是一个获取对象类型的函数，当我们查看某个对象的类型时可以使用。
如想获取 a 的类型我们可以这样使用：
type(a)

  1. 如果a 是一个整数则返回
  2. 如果a 是一个浮点数则返回
  3. 如果a 是一个字符串则返回
  4. 如果a 是一个函数则返回
  5. 如果a 是一个元组则返回
  6. 如果a 是一个字典则返回
  7. 如果a 是一个模块则返回
    … 等等

整数的表示方法

首先我们可以按进制区分：

1. 10进制
2. 2进制
3. 16进制 （0x8af34f 以0x开头)
4. 8进制

浮点数

浮点数就是通常说的带小数点的数字。
如：23.2，0.12，12e3,12e-4
12e3 = 1200.0
12e-4 = 0.0012
带e的表示方法是科学记数法

字符串

Python 中的字符串可以用’’、””、’’’ ‘’’、””” “””四种表示方法，最后两种效果相同。
当我们要在字符串中使用引号时需要进行转义，如下：
‘I\’m \”OK\”!’

\’ \” 代表转义成’和”
还有\n 代表换行

‘’’ ‘’’ 三个引号的字符串

‘’’hello world
I’m “OK”
end
‘’’

将会打印出引号里边的格式，3个引号内部的所有内容将会原封不动的输出。
在字符串前加上r 代表不进行转义，如遇到\n将不会换行，而是最为字符串输出。

布尔值

布尔值只有True(真)和False(假),在Python中我们可以直接使用True 和 False代表真和假。要注意大小写。

布尔值可以使用 and,or,not进行运算：
and 是与的关系只有参与运算的所有布尔值都为真时结果才为真。
or 只要参与运算的所有布尔值中有一个为真则结果就为真。
not 取反

空值 None

空值是Python里一个特殊的值，用None表示。None不代表任何数据，就是空。

变量

变量很好理解，就是可以变化的量，与之对应的是常量，常量是不能改变的量。

a = ‘hello’
b = a
a = ‘world’

  1. Python首先在内存中创建了一个字符串’hello’,然后在内存中创建了一个名为a的变量，并且指向字符串’hello’。
  2. Python在内存中创建变量b，并且把b也指向’hello’
  3. Python在内存中创建字符串’world’,并且把a 指向它。
    这时b = ‘hello’, a = ‘world’

除法

Python有几种除法：

  1. 10/3 精确除法
  2. 10 // 3 整除只保留结果的整数部分
  3. 10%3 求余 只保留余数

认识C enum 枚举

枚举类型 enum 是用来定义一组整型常量的模板。枚举量默认从零开始，我们也可以改变枚举两从指定的地方开始。

定义枚举

enum Day {Mon,Tues,Wed,Thurs,Fri,Sat,Sun}

这样我们就定义了一个枚举类型 Day; 定义之后我们就可以使用 Day 来定义枚举对象了，如下：

enum Day a;
a = Tues; // a 这时等于1
